

# [PRD] 프리미엄 Cloud & AI 인프라 솔루션 랜딩 페이지

## 1. 프로젝트 개요

* **서비스명:** (가칭) Cloud-Ops Deep Insight
* **목적:** 기업 대상의 **Cloud 인프라, DevOps, RAG, AIOps, MLOps** 구축 역량을 홍보하고 잠재 고객 문의를 확보
* **핵심 타겟:** 안정적인 클라우드 전환 및 AI 시스템(RAG) 도입을 원하는 기업 IT 담당자
* **핵심 기술 스택:**
* **Frontend:** React, Tailwind CSS, Framer Motion
* **Backend:** Java Spring Boot, Spring Data JPA
* **Database:** PostgreSQL

## 3. 핵심 기능 (Core Scope)

1. **클라우드 & 오퍼레이션 벤토 쇼케이스:** 클라우드 인프라와 AI Ops 등 5개 핵심 역량을 비대칭 그리드 카드로 시각화.
2. **스마트 문의 시스템 (Inquiry):** 고객이 문의 유형(Cloud RAG, DevOps 등)을 선택하면 DB에 저장되고 내 이메일로 자동 전송.
3. **게시판 시스템:** 관리자 페이지에서 실시간으로 업데이트 가능한 공지사항 및 채용 게시판.
4. **이미지 및 콘텐츠 관리:** 관리자가 메인 페이지의 핵심 이미지나 텍스트를 직접 수정할 수 있는 백오피스.

## 4. 상세 화면별 UI 구성

### 4.1. 페이지 1: 메인 페이지 (Main Showcase)

* **UI 요소:**
* **Hero Section:** "Architecting the Future of Cloud & RAG" 헤드라인. 배경에는 은은한 클라우드 입자 애니메이션.
* **Bento Grid (핵심 역량):**

* **Board List:** 최근 공지사항과 채용 소식을 텍스트 리스트 형태로 깔끔하게 노출.



### 4.2. 페이지 2: 문의하기 (Contact Modal)

* **UI 요소:**
* **서비스 선택 (Select Box):** [클라우드 RAG 구축, 데브옵스, AI옵스, ML옵스, 기타] 중 선택.

### 4.3. 페이지 3: 관리자 페이지 (Admin Dashboard)

* **UI 요소:**
* **로그인:** 관리자 인증 후 접근.
* **문의 목록:** 고객이 남긴 문의 내역을 PostgreSQL에서 불러와 리스트로 출력.
* **게시물 관리:** 공지사항/채용 글쓰기, 수정, 삭제 및 이미지 업로드 기능.

## 5. 백엔드 및 DB 구조 (Spring Boot + PostgreSQL)

### **데이터베이스 (PostgreSQL)**

1. **Inquiry Table:** 고객의 서비스 선택 유형과 연락처 정보를 저장.
2. **Post Table:** 공지사항 및 채용 관련 데이터(제목, 본문, 이미지 경로) 저장.

### **백엔드 로직**

* **REST API:** 프론트엔드와 JSON 형식으로 데이터 통신.
* **Email Service:** Spring Boot의 `JavaMailSender`를 연동하여 문의 접수 시 사용자님의 이메일로 즉시 알림 발송.

---

## 6. 사용자 흐름 (User Flow)

1. **접속:** 사용자가 다크 테마의 전문적인 메인 페이지에서 기업의 **Cloud/Ops 역량**을 확인.
2. **문의 결정:** 관심 있는 서비스 카드를 보고 '문의하기' 클릭.
3. **데이터 처리:** 입력된 정보가 **Spring Boot**를 거쳐 **PostgreSQL**에 저장되고, 동시에 관리자(나)에게 이메일 도착.
4. **정보 업데이트:** 관리자가 새로운 채용 소식이나 인프라 사례를 올리면 메인 페이지의 벤토 그리드 하단에 실시간 반영.

---

## 7. 향후 추가 기능 (Backlog)
